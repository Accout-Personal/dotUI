## Usage

Use a `TextField` to allow users to input custom text entries with a keyboard.

- Always include a label (it can be visible or hidden with `aria-label`)

## Options

### Variants

Use the `variant` prop to control the visual style of the `TextField`. The default variant is `default`.

<ComponentPreview name="demos/components/core/text-field/variants" className="max-w-[250px]" />

### Sizes

Use the `size` prop to control the size of the `TextField`. The default variant is `md`.

<ComponentPreview name="demos/components/core/text-field/sizes" className="max-w-lg" />

### Prefix and suffix

To add additional context for the `TextField`, use the `prefix` and `suffix` props.

### Label

A visual label can be provided for the `TextField` using the label prop, or a hidden label using `aria-label` prop.

<ComponentPreview name="demos/components/core/text-field/label" className="max-w-[250px]" />

### Description

A description can be supplied to a TextField via the `description` prop. The description is always visible unless the `validationState` is `“invalid”` and an error message is provided.

<ComponentPreview name="demos/components/core/text-field/description" className="max-w-[250px]" />

### Contextual help

A ContextualHelp element may be placed next to the label to provide additional information or help about a TextField.

  name="demos/components/core/text-field/contextual-help"

### Error message

An error message can be supplied to a TextField, which will be displayed when the `validationState` is `“invalid”`.

<ComponentPreview name="demos/components/core/text-field/error-message" className="max-w-[250px]" />

### Loading

Use the `isLoading` prop to control the loading state of the TextField.
Use the `loaderPosition` prop to control the position of the loader.

<ComponentPreview name="demos/components/core/text-field/loading" className="max-w-xs" />

### Disabled

Use the `isDisabled` prop to disable the TextField.

<ComponentPreview name="demos/components/core/text-field/disabled" className="max-w-[250px]" />

### ReadOnly

The `isReadOnly` boolean prop makes the TextField's text content immutable. Unlike isDisabled, the TextField remains focusable and the contents can still be copied.

<ComponentPreview name="demos/components/core/text-field/read-only" className="max-w-[250px]" />

### Required

Use the `isRequired` prop to mark the TextField as required.
Use the `necessityIndicator` prop to control the visual style of the required state.

<ComponentPreview name="demos/components/core/text-field/required" className="max-w-sm" />

## API Reference

### TextField

This component inherits and merges props from TextFieldRoot and Field parts.

| Prop             | Type                                                           | Default     | Description                                  |
| ---------------- | -------------------------------------------------------------- | ----------- | -------------------------------------------- |
| `variant`        | `"default" \| "primary" \| "danger" \| "success" \| "warning"` | `"default"` | The visual style of the TextField.           |
| `size`           | `"sm" \| "md" \| "lg"`                                         | `"md"`      | The size of the input.                       |
| `prefix`         | `React.ReactNode`                                              | -           | A visual to display before the input.        |
| `suffix`         | `React.ReactNode`                                              | -           | A visual to display after the input.         |
| `isLoading`      | `boolean`                                                      | -           | Whether the TextField is in a pending state. |
| `loaderPosition` | `"prefix" \| "suffix"`                                         | -           | The loader position                          |

### TextFieldRoot

| Prop                 | Type                                                                                                   | Default  | Description                                                                                                                                                                                                                           |
| -------------------- | ------------------------------------------------------------------------------------------------------ | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `isInvalid`          | `boolean`                                                                                              | -        | Whether the value is invalid.                                                                                                                                                                                                         |
| `isDisabled`         | `boolean`                                                                                              | -        | Whether the input is disabled.                                                                                                                                                                                                        |
| `isReadOnly`         | `boolean`                                                                                              | -        | Whether the input can be selected but not changed by the user.                                                                                                                                                                        |
| `isRequired`         | `boolean`                                                                                              | -        | Whether user input is required on the input before form submission.                                                                                                                                                                   |
| `validate`           | `(value: string) => ValidationError \| true \| null \| undefined`                                      | -        | A function that returns an error message if a given value is invalid. Validation errors are displayed to the user when the form is submitted if validationBehavior="native". For realtime validation, use the isInvalid prop instead. |
| `autoFocus`          | `boolean`                                                                                              | -        | Whether the element should receive focus on render.                                                                                                                                                                                   |
| `value`              | `string`                                                                                               | -        | The current value (controlled).                                                                                                                                                                                                       |
| `defaultValue`       | `string`                                                                                               | -        | The default value (uncontrolled).                                                                                                                                                                                                     |
| `autoComplete`       | `string`                                                                                               | -        | Describes the type of autocomplete functionality the input should provide if any.                                                                                                                                                     |
| `maxLength`          | `number`                                                                                               | -        | The maximum number of characters supported by the input.                                                                                                                                                                              |
| `minLength`          | `number`                                                                                               | -        | The minimum number of characters required by the input.                                                                                                                                                                               |
| `pattern`            | `string`                                                                                               | -        | Regex pattern that the value of the input must match to be valid.                                                                                                                                                                     |
| `type`               | `'text' \| 'search' \| 'url' \| 'tel' \| 'email' \| 'password' \| string & {}`                         | -        | The type of input to render.                                                                                                                                                                                                          |
| `inputMode`          | `'none' \| 'text' \| 'tel' \| 'url' \| 'email' \| 'numeric' \| 'decimal' \| 'search'`                  | -        | Hints at the type of data that might be entered by the user while editing the element or its contents.                                                                                                                                |
| `name`               | `string`                                                                                               | -        | The name of the input element, used when submitting an HTML form.                                                                                                                                                                     |
| `validationBehavior` | `'native' \| 'aria'`                                                                                   | 'native' | Whether to use native HTML form validation to prevent form submission when the value is missing or invalid, or mark the field as required or invalid via ARIA.                                                                        |
| `children`           | `ReactNode \| (values: TextFieldRenderProps & {defaultChildren: ReactNode \| undefined}) => ReactNode` | -        | The children of the component. A function may be provided to alter the children based on component state.                                                                                                                             |
